\Symmetry

\SymmetryResponse

The Response object holds the necessary file path details to map to a view file or instruction set.

Multiple Response objects can be nested within each other.

display()

display(string  $top = false, boolean  $headers = false) : \Symmetry\multitype:|string

This will recursively process Response objects and any other connected parameter.

Output data is organized in arrays, all array keys will become variable names with the array values becoming the variable values. data['key']='value' translates to $key='value' within the included script file.

capture()

capture(boolean  $start = true, string  $position = "top", boolean  $append = true)

Takes a portion of a view script to store for later output.

View scripts may be rapidly developed by putting css and js details in with the php, by so doing css and js could be littered throughout the final output. By capturing css and js, those bits can be output in one spot, prefereably in a layout.

Parameters

boolean $start

a start flag to indicate being turned on.
string $position

a label describing the bin the captured data goes into
boolean $append

whether to put data at the end of the beginning of the bin.
